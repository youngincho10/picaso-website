# PICASO SAY × KNOK 부스 웹사이트 — 실제 입력 버전

이번 버전은 자동으로 만들어지는 데모 결과를 제거하고, 실제 입력이 들어온 경우에만 결과가 표시되도록 수정한 버전입니다.

## 핵심 변경

### SAY

- 미리 적힌 문장을 누르는 방식 제거
- 키보드 직접 입력 제거
- 마이크 버튼을 누른 뒤 **실제로 말한 음성만** 사용
- Chrome 음성 인식 결과를 실시간 표시
- 음성 인식이 끝나야 SAY 전달 문장을 생성
- 마이크가 아무 음성도 듣지 못하면 결과를 생성하지 않음
- 인식 전 문장과 전달 문장을 화면에서 구분
- 결과를 한국어 음성으로 출력

주의: 현재 SAY의 음성 인식은 Chrome의 Web Speech Recognition을 사용합니다. 실제 구음장애 음성에 특화된 별도 AI 모델은 아직 연결되어 있지 않습니다. 이후 Gemini API 또는 자체 모델을 연결할 때 `enhancePhrase()` 부분을 교체하면 됩니다.

### KNOK

- 센서 없이 자동으로 랜덤 측정하던 기능 완전 제거
- 공개 화면의 데모 측정 버튼 제거
- ESP32-S3가 연결되어야 측정 버튼 활성화
- 측정 시작 후 **실제 충격 이벤트**를 기다림
- 10초 안에 충격 이벤트가 없으면 측정값을 만들지 않고 종료
- 일반 숫자, 부팅 메시지, 연속 센서값은 측정 결과로 사용하지 않음
- 아래처럼 명시적인 충격 이벤트만 인식

```text
HIT_SCORE:54
IMPACT_SCORE:54
HIT_RAW:1650
IMPACT_RAW:1650
{"event":"impact","score":54}
{"event":"impact","raw":1650}
```

`54`, `1650`, `READY`, `CONNECTED`, 일반 로그, `type:"impact"`처럼 위 형식과 정확히 일치하지 않는 데이터는 결과로 사용하지 않습니다. 측정 시작 전에 수신한 이벤트와 한 측정이 끝난 직후 1초 동안 들어오는 중복 이벤트도 무시합니다.

## 파일

- `index.html` — 화면
- `styles.css` — 디자인

## SAY 사용

1. SAY 화면으로 들어갑니다.
2. 일상·학교·병원 상황을 선택합니다.
3. **말하기 시작**을 누릅니다.
4. Chrome에서 마이크 권한을 허용합니다.
5. 직접 말합니다.
6. 마이크가 들은 문장이 왼쪽에 표시됩니다.
7. 인식이 끝난 후 SAY 전달 문장이 오른쪽에 표시됩니다.
8. **음성으로 전달**을 누르면 문장을 읽습니다.

음성 인식은 데스크톱 Chrome과 인터넷 연결이 필요할 수 있습니다.

> Codex 내부 브라우저에서는 마이크 권한이 `허용됨`으로 보여도 Chrome Web Speech 인식 서비스가 차단될 수 있습니다. 행사 체험은 반드시 **독립적으로 실행한 데스크톱 Google Chrome**의 `http://localhost:8000`에서 진행하세요.

### Chrome 마이크 권한 설정

1. `http://localhost:8000`을 연 상태에서 주소창 왼쪽의 사이트 설정 아이콘을 누릅니다.
2. **마이크 → 허용**으로 설정합니다.
3. 사용할 마이크가 맞는지 확인하고 페이지를 새로고침합니다.
4. macOS가 차단했다면 **시스템 설정 → 개인정보 보호 및 보안 → 마이크 → Google Chrome**을 허용합니다.

화면에는 권한 확인 중·허용됨·요청 필요·거부됨·마이크 없음·듣는 중·음성 없음·완료·브라우저 미지원 상태가 구분되어 표시됩니다.

## KNOK 사용

1. ESP32-S3에 `knok_esp32_example.ino`를 업로드합니다.
2. 센서의 아날로그 출력 핀과 `SENSOR_PIN`을 맞춥니다.
3. ESP32-S3를 USB로 맥북에 연결합니다.
4. KNOK 화면에서 **KNOK 기기 연결**을 누릅니다.
5. ESP32 포트를 선택합니다.
6. **측정 시작**을 누릅니다.
7. 체험판을 실제로 한 번 두드립니다.
8. 센서가 충격 기준을 넘었을 때만 측정값이 표시됩니다.

## 센서 보정

설정 화면에서 다음 값을 조정합니다.

- 목표 최솟값·최댓값: 기본 50–60
- RAW 최소·최대: 실제 센서 충격값을 0–100으로 변환하는 기준
- ESP32의 `TRIGGER_DELTA`: 충격으로 인정할 최소 변화량

처음에는 Arduino 시리얼 모니터에서 여러 번 두드려 `HIT_RAW` 값을 확인한 뒤 보정하는 것이 안전합니다.

`knok_esp32_example.ino` 상단에서 다음 값을 조정합니다.

- `SENSOR_PIN`: 센서 아날로그 출력과 실제로 연결한 ESP32-S3 ADC 핀
- `TRIGGER_DELTA`: 평상시 기준값에서 얼마나 벗어나야 충격으로 인정할지
- `COOLDOWN_MS`: 한 번의 충격이 여러 번 전송되지 않도록 막는 시간
- `PEAK_WINDOW_MS`: 충격 직후 최고값을 찾는 짧은 측정 구간

Arduino IDE에서 ESP32-S3 보드와 포트를 선택한 뒤 예제 코드를 업로드합니다. 시리얼 속도는 `115200`입니다. 평상시에는 연속 센서값을 전송하지 않고, 기준치를 넘는 순간에만 `HIT_RAW:숫자`를 한 줄 전송합니다.

## 행사 전 확인

- Chrome 마이크 권한 허용
- SAY를 실제 목소리로 10회 이상 시험
- KNOK를 약함·보통·강함으로 각각 10회 이상 시험
- 주변 말소리만으로 KNOK 충격 이벤트가 발생하지 않는지 확인
- 체험판을 두드리지 않았을 때 측정값이 생기지 않는지 확인
- 맥북 자동 잠금 끄기
- 충전기 연결
- 알림 끄기
- 전체 화면 실행

## 현재 기술적 한계

- SAY는 구음장애 전용으로 학습된 모델이나 의료용 시스템이 아니라 Chrome의 일반 Web Speech Recognition 결과를 제한적으로 정리하는 체험 버전입니다.
- Chrome 음성 인식은 네트워크 상태와 운영체제의 마이크 권한에 영향을 받을 수 있습니다.
- KNOK의 0–100 값은 설정한 RAW 범위로 환산한 체험용 충격 지수이며 인증 계측값이나 법적 증거가 아닙니다.
- 센서 종류, 부착 위치, 체험판 재질에 따라 `SENSOR_PIN`, `TRIGGER_DELTA`, RAW 최소·최대 값을 현장에서 다시 보정해야 합니다.
- 실제 ESP32-S3와 센서가 없으면 브라우저의 연결·분리와 물리 충격 감지 전체 흐름을 완전히 검증할 수 없습니다.

## SAY의 어눌한 발화 보완 범위

- SAY는 반드시 마이크로 실제 음성이 들어온 뒤에만 결과를 만듭니다.
- Chrome이 반환한 최대 5개의 음성 인식 후보를 비교합니다.
- 현재 선택한 상황(일상·학교·병원), 핵심 단어, 문장 유사도를 함께 사용해 가장 가능성 높은 후보를 선택합니다.
- `무 마시고 시퍼요`, `도아주세요`, `잠시 기달려 주세여`처럼 일부 발음이 흐려진 표현은 보완 규칙으로 정리할 수 있습니다.
- 다만 현재 버전은 구음장애 음성에 특화해 학습한 모델이 아니라 Chrome의 일반 음성 인식을 사용합니다. 발음이 많이 불명확해 Chrome이 핵심 단어를 전혀 잡지 못하면 정확히 복원할 수 없습니다.
- 실제 제품 수준의 정확도를 내려면 구음장애 음성 데이터로 학습한 모델이나 Gemini·별도 음성 모델을 연결해야 합니다.

화면에는 직접 말해볼 수 있는 세 가지 예시가 표시되지만, 예시는 버튼이 아니며 자동 입력되지 않습니다.

## ESP32-S3 SuperMini + I2S MEMS 마이크 연결

knok_esp32_example.ino는 INMP441 호환 I2S MEMS 마이크를 실제 충격음 센서로 사용합니다.

| MEMS 마이크 | ESP32-S3 SuperMini |
| --- | --- |
| VDD | 3.3V |
| GND | GND |
| SCK / BCLK | GPIO 12 |
| WS / LRCL | GPIO 11 |
| SD / DOUT | GPIO 10 |
| L/R | GND (Left 채널) |

1. PlatformIO에서 Archive 폴더를 프로젝트로 열거나 pio run -d Archive -t upload --upload-port COM번호를 실행합니다.
2. 데스크톱 Chrome에서 KNOK 화면의 KNOK 기기 연결을 누르고 ESP32-S3의 USB 포트를 선택합니다.
3. 측정 시작을 누르면 웹이 ARM을 전송하고 ESP32가 실제 마이크 충격음을 한 번 감지합니다.
4. 실제 피크는 JSON impact 이벤트로 전송되어 KNOK 게이지와 최근 측정에 표시됩니다.
5. 너무 민감하면 MIN_TRIGGER_PEAK 또는 NOISE_PEAK_MULTIPLIER를 올리고, 둔하면 낮춥니다.

L/R 핀이 3.3V에 연결되면 Right 채널이므로 현재 Left 채널 설정에서는 신호가 잡히지 않습니다. 반드시 L/R을 GND에 연결하세요. 측정값은 음향 기반 체험용 충격 지수이며 인증 소음계나 법적 증거용 계측값이 아닙니다.
